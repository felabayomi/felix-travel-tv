# Auto-Play Decision Tree Demo — Build Pattern

A reusable pattern for creating an infinitely looping, auto-playing walkthrough of any decision tree. Built with React, TypeScript, and Framer Motion. Designed for livestreams, kiosks, trade show displays, or any unattended demo scenario.

---

## What It Does

- Starts at the root node of a decision tree
- Waits a few seconds so the viewer can read the question
- Highlights a randomly chosen answer with a glow animation
- Advances to the next node after a short pause
- When a terminal result is reached, shows a result card with a call to action
- A countdown bar ticks down, then the whole loop restarts with a fresh random path
- Loops infinitely — no user interaction required

---

## Core Concept: The State Machine

The demo operates as a simple two-phase state machine:

```
"question" phase
  → waits QUESTION_PAUSE_MS
  → picks random option, sets highlightIndex
  → waits HIGHLIGHT_MS
  → if option has resultId → switch to "result" phase
  → if option has nextNode → stay in "question" phase, update nodeId

"result" phase
  → displays result card for RESULT_SHOW_MS
  → resets back to root node → "question" phase (round increments)
```

## Required Data Shape

Your decision tree needs two data structures:

### 1. The Tree (nodes)

```ts
interface QuizOption {
  label: string;
  iconName: string;        // optional — any icon library
  nextNode?: string;       // ID of the next question node
  resultId?: string;       // ID of the terminal result (leaf node)
}

interface QuizNode {
  id: string;
  question: string;
  description?: string;
  options: QuizOption[];
}

// Each option must have EITHER nextNode OR resultId — not both, not neither
const TREE: Record<string, QuizNode> = {
  "root": {
    id: "root",
    question: "What brings you here today?",
    options: [
      { label: "Option A", iconName: "Star", nextNode: "node_a" },
      { label: "Option B", iconName: "Heart", resultId: "result_b" },
    ]
  },
  "node_a": {
    id: "node_a",
    question: "Tell me more...",
    options: [
      { label: "Choice 1", iconName: "Sparkles", resultId: "result_1" },
      { label: "Choice 2", iconName: "Gem",      resultId: "result_2" },
    ]
  }
};
```

### 2. The Results (leaf nodes)

```ts
interface QuizResult {
  id: string;
  name: string;
  description: string;
  bestFor: string;       // "Tag A · Tag B · Tag C" — split on · for badges
  runnerUp?: string;     // optional: ID of a secondary recommendation
}

const RESULTS: Record<string, QuizResult> = {
  "result_1": {
    id: "result_1",
    name: "Recommended Thing",
    description: "Why this is the best match...",
    bestFor: "Use Case A · Audience B · Scenario C",
    runnerUp: "result_2"
  }
};
```

## Timing Constants

Three constants control all pacing. Adjust to taste:

```ts
const QUESTION_PAUSE_MS = 4500;  // Time to read the question before any highlight
const HIGHLIGHT_MS      = 3200;  // Time the chosen answer glows before advancing
const RESULT_SHOW_MS    = 14000; // Time the result card is shown before restarting
```

**Recommended ranges:**
- Fast (kiosk): `2000 / 1500 / 8000`
- Standard (demo): `3200 / 2400 / 10000`
- Slow (livestream): `4500 / 3200 / 14000`

## The State Object

```ts
interface DemoState {
  phase: "question" | "result";
  nodeId: string;           // current node ID in the tree
  highlightIndex: number | null;  // which option is glowing (null = none)
  resultId: string | null;  // set when a terminal result is reached
  path: string[];           // labels of answers chosen so far (for breadcrumb)
  stepKey: number;          // increment to re-trigger the question useEffect
  round: number;            // which loop iteration we're on
}
```

## The Two useEffect Hooks

All auto-play logic lives in two effects. No event handlers needed.

### Effect 1 — Drive the question phase

```ts
useEffect(() => {
  if (state.phase !== "question") return;
  const node = TREE[state.nodeId];
  if (!node) return;

  let t2: ReturnType<typeof setTimeout>;

  // Step 1: Wait, then highlight a random option
  const t1 = setTimeout(() => {
    const idx = Math.floor(Math.random() * node.options.length);
    setState(s => ({ ...s, highlightIndex: idx }));

    // Step 2: Wait, then advance
    t2 = setTimeout(() => {
      const chosen = node.options[idx];
      const newPath = [...stateRef.current.path, chosen.label];

      if (chosen.resultId) {
        // Terminal — switch to result phase
        setState(s => ({ ...s, phase: "result", resultId: chosen.resultId!, path: newPath, highlightIndex: null }));
      } else if (chosen.nextNode) {
        // Continue — move to next node
        setState(s => ({ ...s, nodeId: chosen.nextNode!, path: newPath, highlightIndex: null, stepKey: s.stepKey + 1 }));
      }
    }, HIGHLIGHT_MS);
  }, QUESTION_PAUSE_MS);

  // Cleanup both timers if the effect re-runs or component unmounts
  return () => { clearTimeout(t1); clearTimeout(t2); };

}, [state.phase, state.nodeId, state.stepKey]);
// ↑ stepKey increments on every node change so the effect re-fires even if nodeId stays the same
```

> **Why `stateRef`?** The inner `t2` callback closes over the path value from when `t1` fired. Using a ref (`stateRef.current = state`) ensures it reads the latest state rather than a stale closure.

```ts
const stateRef = useRef(state);
stateRef.current = state; // update every render
```

### Effect 2 — Drive the result phase

```ts
useEffect(() => {
  if (state.phase !== "result") return;

  const t = setTimeout(() => {
    setState(s => ({
      phase: "question",
      nodeId: "root",           // ← your root node ID
      highlightIndex: null,
      resultId: null,
      path: [],
      stepKey: s.stepKey + 1,
      round: s.round + 1,
    }));
  }, RESULT_SHOW_MS);

  return () => clearTimeout(t);

}, [state.phase, state.resultId]);
// ↑ resultId in deps ensures this fires fresh for each new result
```

## Highlight Animation (Framer Motion)

The selected option gets a scale bump and a radial glow overlay:

```tsx
<motion.div
  animate={{ scale: isHighlighted ? 1.035 : 1 }}
  transition={{ type: "spring", stiffness: 320, damping: 28 }}
  className={isHighlighted ? "border-primary shadow-xl" : "border-border"}
>
  {isHighlighted && (
    <motion.div
      layoutId="selector"
      className="absolute inset-0 rounded-xl"
      style={{ background: "radial-gradient(ellipse at 30% 50%, rgba(14,116,144,0.12), transparent 70%)" }}
    />
  )}
  {/* option content */}
</motion.div>
```

`layoutId="selector"` makes the glow smoothly jump between options across re-renders.

## Countdown Bar

The progress bar on the result card uses a single `scaleX` animation tied to `RESULT_SHOW_MS`:

```tsx
<div className="h-1.5 bg-border/30 rounded-full overflow-hidden">
  <motion.div
    className="h-full bg-primary rounded-full"
    initial={{ scaleX: 0 }}
    animate={{ scaleX: 1 }}
    transition={{ duration: RESULT_SHOW_MS / 1000 - 0.75, ease: "linear", delay: 0.75 }}
    style={{ transformOrigin: "left" }}
  />
</div>
```

The `delay: 0.75` matches the staggered fade-in of the result card content so the bar starts after everything has appeared.

## Themed Round Labels

Instead of "Round 1 / Round 2", cycle through a list of themed labels:

```ts
const ROUND_THEMES = [
  "Bon Voyage", "Adventure Awaits", "Where To Next?",

// In JSX:
{ROUND_THEMES[(state.round - 1) % ROUND_THEMES.length]}
```

## Path Breadcrumb

Track every label the demo "chose" and render it at the bottom so viewers can follow the logic:

```tsx
{state.path.map((label, i) => (
  <span key={i}>
    {i > 0 && <span>›</span>}
    <span className={i === state.path.length - 1 ? "font-semibold text-primary" : ""}>
      {label}
    </span>
  </span>
))}
```

The last item is highlighted so it's clear what was just selected.

## Adapting to a New Decision Tree

1. **Define your nodes** — replace the `TREE` record with your own questions and branching logic
2. **Define your results** — replace `RESULTS` with your terminal outcomes
3. **Set the root node ID** — update the `"q1_trip_type"` references to your root node ID
4. **Adjust timing** — tune the three constants for your content length and audience
5. **Customize the result card** — swap in whatever fields your results have (the pattern only requires `name` and `description` at minimum)
6. **Customize the CTA** — replace the phone number / URL with your own contact details
7. **Customize the themed labels** — replace `ROUND_THEMES` with phrases relevant to your domain

The state machine, effects, and animations are fully content-agnostic — only the data and copy need to change.
